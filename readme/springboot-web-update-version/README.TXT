升级脚本执行顺序：
升级包分别上传到Mysql,Nginx,PortalWeb1,PortalWeb2所在的服务器的/root/portal_update/ 目录，
如果升级包目录不存在，创建这个目录:  mkdir /root/portal_update/

在mysql所在服务器执行升级mysql的脚本
在PortalWeb1所在服务器执行升级PortalWeb1的脚本
在PortalWeb2所在服务器执行升级PortalWeb2的脚本(集群模式下，不需要升级PortalWeb2，每台都只有PortalWeb1)
在Nginx所在服务器执行升级Nginx的脚本

------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------
【升级步骤】

***！！！ 确认现场版本是否是 V1.1.03201或者是其之后的版本 是的话再升级
------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------

1、升级Mysql脚本
	【山西地区】 执行	sh update1_Mysql_SX.sh
	【深圳地区】 执行	sh update1_Mysql_SZ.sh
    【江苏地区】 执行	sh update1_Mysql_JS.sh
    【其他地区】 执行	sh update1_Mysql.sh	

2、升级PortalWeb1	
	sh update2_PortalWeb1.sh	（如果PortalWeb部署在多台服务器，那么每台都要执行）

   （升级PortalWeb1 ,启动服务的时间大概20秒左右， tail -f /u01/PortalWeb1/logs/PortalWeb.out  看下日志是否启动成功，PortalWeb1启动成功之后再升级nginx）

3、升级Nginx
	sh update3_Nginx.sh	（如果Nginx部署在多台服务器上，每台都要执行）

4、升级PortalWeb2
	sh update4_PortalWeb2.sh	（如果集群模式部署，跳过。每台服务器上都是PortalWeb1服务，不需要升级PortalWeb2服务）
